# Contributing (Solo-team Rules)

## Branch policy
- `main` is protected (no direct pushes).
- All changes go through PRs.

## Branch naming
- `day0_5/<short>`, `day1/<short>`, `fix/<short>`, `exp/<short>`

## Commit message format
- `feat(gfx): Day0.5 - ...`
- `fix(app): Day1 - ...`
- `docs(contracts): Day0.5 - ...`
- `chore(repo): ...`
- `ci: ...`

## PR Definition of Done
- Build green (VS2022 x64)
- Contract updated: `docs/contracts/DayX.md`
- Evidence recorded: log + screenshot/capture reference
- No “nice-to-have” refactor unless needed to keep build green
