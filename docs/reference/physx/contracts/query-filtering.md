- Outputs:

- Edge cases:
  - Cached shapes bypass filtering and are assumed blocking if a cached hit is found; public raycast and sweep docs warn that using past touching hits as cache is incorrect because cached hits are treated as blocking (`.ref/PhysX_4.0/physx/include/PxScene.h:1311`, `.ref/PhysX_4.0/physx/include/PxScene.h:1312`, `.ref/PhysX_4.0/physx/include/PxScene.h:1313`, `.ref/PhysX_4.0/physx/include/PxScene.h:1341`, `.ref/PhysX_4.0/physx/include/PxScene.h:1342`, `.ref/PhysX_4.0/physx/include/PxScene.h:1343`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:393`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:394`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:404`).
  - Overlap queries require a touch buffer unless `eANY_HIT` is set; the implementation checks this before traversal (`.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:684`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:709`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:711`).
- Rejection/filtering rules:
  - `eNONE` from prefilter or postfilter suppresses the candidate; implementation returns early after prefilter `eNONE` and asserts the remaining postfilter fallthrough as `eNONE` (`.ref/PhysX_4.0/physx/include/PxQueryFiltering.h:91`, `.ref/PhysX_4.0/physx/include/PxQueryFiltering.h:112`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:408`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:409`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:560`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:561`).
- Ordering/tie-break rules:
  - Filter callbacks are not guaranteed to run in ray or sweep distance order; public docs state callbacks are not sorted, and batched shader docs state processing follows traversal candidate order (`.ref/PhysX_4.0/physx/include/PxQueryFiltering.h:165`, `.ref/PhysX_4.0/physx/include/PxQueryFiltering.h:247`, `.ref/PhysX_4.0/physx/include/PxQueryFiltering.h:248`).
  - Touch hits are stored only when their hit distance is less than or equal to the current shrunk block distance; block hits shrink that distance and update `block/hasBlock` (`.ref/PhysX_4.0/physx/include/PxQueryFiltering.h:93`, `.ref/PhysX_4.0/physx/include/PxQueryFiltering.h:94`, `.ref/PhysX_4.0/physx/include/PxQueryFiltering.h:95`, `.ref/PhysX_4.0/physx/include/PxQueryReport.h:268`, `.ref/PhysX_4.0/physx/include/PxQueryReport.h:269`, `.ref/PhysX_4.0/physx/include/PxQueryReport.h:278`, `.ref/PhysX_4.0/physx/include/PxQueryReport.h:279`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:503`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:541`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:549`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:553`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:556`, `.ref/PhysX_3.4/PhysX_3.4/Source/PhysX/src/NpSceneQueries.cpp:557`).
